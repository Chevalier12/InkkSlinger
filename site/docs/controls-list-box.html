ListBox

ListBox is the selectable vertical items control for menu-like lists, inspectors, pickers, and scrolling collections of rows. It combines the ItemsControl item pipeline with Selector selection state and an internal ScrollViewer, then keeps realized ListBoxItem containers synchronized with hover and selection.

Quick start

Use ListBox when you need a vertically stacked collection of selectable rows with built-in scrolling and a conventional selected-item API.

Minimal single-select list

<ListBox Width="260"
         Height="180"
         SelectedIndex="0">
  <ListBoxItem>
    <Label Content="Inbox" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Drafts" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Archive" />
  </ListBoxItem>
</ListBox>

Populate simple text items in code

var listBox = new ListBox
{
    Width = 320,
    Height = 220
};

listBox.Items.Add("Alpha");
listBox.Items.Add("Beta");
listBox.Items.Add("Gamma");
listBox.SelectedIndex = 1;

Enable multi-select toggling

<ListBox Width="300"
         Height="200"
         SelectionMode="Multiple">
  <ListBoxItem>
    <Label Content="Shaders" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Textures" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Audio" />
  </ListBoxItem>
</ListBox>

Default recommendation: use plain string items for simple lists, and move to explicit ListBoxItem markup or ItemContainerStyle when you need custom row chrome, hover treatment, or selected-state visuals.

Items and container model

ListBox inherits from Selector, which in turn inherits from ItemsControl. That means you get the normal Items, ItemsSource, and ItemContainerStyle pipeline, but the interactive row contract still centers on ListBoxItem.

Explicit containers

If you supply ListBoxItem elements directly, the control uses them as-is. This is the clearest way to author richer row content and keep full control over selection visuals.

Generated containers

If you add plain objects, ListBox creates ListBoxItem containers for them. Raw UIElement items are hosted as the container content, and non-visual values fall back to a generated Label whose content is ToString().

  • Items and ItemsSource follow the normal ItemsControl rule: do not mutate Items while ItemsSource is set.
  • ItemContainerStyle is the primary row-level styling hook for generated containers.
  • Generated label rows copy the list box typography fast-path for FontFamily, FontSize, FontWeight, and FontStyle.

Bind a simple string collection

<ListBox Width="320"
         Height="220"
         ItemsSource="{Binding RecentProjects}"
         SelectedItem="{Binding ActiveProject}" />

That works best when the bound items are strings or when the bound objects provide a useful ToString() result. For custom row layout, prefer explicit ListBoxItem content or a container style.

Author a richer row explicitly

<ListBox Width="360" Height="220">
  <ListBoxItem>
    <Grid>
      <Grid.ColumnDefinitions>
        <ColumnDefinition Width="Auto" />
        <ColumnDefinition Width="*" />
      </Grid.ColumnDefinitions>

      <Border Width="10" Height="10" Background="#4FC7C1" Margin="0,0,10,0" />
      <StackPanel Grid.Column="1">
        <Label Content="Build agent A" />
        <Label Content="Online" Opacity="0.7" />
      </StackPanel>
    </Grid>
  </ListBoxItem>
</ListBox>

Primary customization surface: if the row itself needs selected or hover visuals, style ListBoxItem rather than only styling the content inside it.

Selection behavior

ListBox exposes the standard selector properties and raises the bubbled SelectionChanged routed event. Pointer selection is handled by clicking a realized ListBoxItem container.

Single mode

SelectionMode="Single" is the default. Clicking a row replaces the current selection and updates SelectedIndex plus SelectedItem.

Multiple mode

SelectionMode="Multiple" toggles the clicked row directly. The full selection is exposed through SelectedIndices, while SelectedIndex and SelectedItem reflect the first selected entry.

  • SelectedIndex = -1 clears the selection.
  • SelectedItem can also drive selection when you already have the backing item instance.
  • SelectedIndices is the inspection API for multi-select state; there is no separate SelectedItems collection.

Observe selection changes in code

listBox.SelectionChanged += (_, _) =>
{
    var firstSelected = listBox.SelectedItem;

    foreach (var index in listBox.SelectedIndices)
    {
        Logger.Write($"Selected row: {index}");
    }
};

Drive selection from code

listBox.SelectedIndex = 3;

if (listBox.SelectedItem is ServerEntry server)
{
    Connect(server);
}

Layout, scrolling, and virtualization

ListBox renders its own background and border, then measures and arranges an internal ScrollViewer inside that border. The items host is a vertical stack panel by default, or a VirtualizingStackPanel when IsVirtualizing is enabled.

Scroll behavior

Horizontal scrolling defaults to Disabled. Vertical scrolling defaults to Auto. LineScrollAmount defaults to 24 device-independent units.

Selection visibility

When the selected index changes, the control automatically scrolls the chosen row fully into view if it is currently above or below the viewport.

  • BorderThickness is a single float value, not a Thickness.
  • The clip rect matches the list box layout slot, so scrolled content stays clipped to the control bounds.
  • IsVirtualizing="True" switches the items host, but the control still behaves like a normal vertically scrolling list.

Virtualized log or event stream

<ListBox Width="420"
         Height="260"
         IsVirtualizing="True"
         VerticalScrollBarVisibility="Auto"
         LineScrollAmount="18"
         ItemsSource="{Binding EventLines}" />

Select a far item and let the control reveal it

for (var i = 0; i < 80; i++)
{
    listBox.Items.Add($"Item {i:000}");
}

listBox.SelectedIndex = 79; // The control scrolls so the last row becomes fully visible.

Property reference

Selection API

SelectedIndex: the primary selected row index, or -1 when nothing is selected.

SelectedItem: the primary selected backing item.

SelectedIndices: read-only access to every selected index.

SelectionMode: Single or Multiple.

Scrolling and host behavior

IsVirtualizing: swaps the item host between a normal vertical stack and a VirtualizingStackPanel.

HorizontalScrollBarVisibility and VerticalScrollBarVisibility: forwarded to the internal ScrollViewer.

LineScrollAmount: positive scroll increment for line-based wheel and button scrolling.

Chrome properties

Background and BorderBrush: Color values used for the list body and stroke.

BorderThickness: non-negative single-width border around the hosted scroller.

Inherited item pipeline

Items, ItemsSource, and ItemContainerStyle come from ItemsControl.

FontFamily, FontSize, FontWeight, and FontStyle help drive generated text rows.

For row-level visuals, the key companion type is ListBoxItem, which exposes IsSelected, IsMouseOver, Background, SelectedBackground, BorderBrush, and Padding.

Patterns and examples

Navigation rail

<ListBox Width="240"
         Height="260"
         SelectedIndex="0">
  <ListBoxItem>
    <Label Content="Overview" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Build output" />
  </ListBoxItem>
  <ListBoxItem>
    <Label Content="Diagnostics" />
  </ListBoxItem>
</ListBox>

Bound string list with selected item binding

<ListBox Width="320"
         Height="220"
         ItemsSource="{Binding BuildSteps}"
         SelectedItem="{Binding ActiveBuildStep}" />

Restyle generated rows with ItemContainerStyle

<ListBox Width="340"
         Height="220"
         ItemsSource="{Binding Agents}">
  <ListBox.ItemContainerStyle>
    <Style TargetType="{x:Type ListBoxItem}">
      <Setter Property="Background" Value="#152230" />
      <Setter Property="SelectedBackground" Value="#28445F" />
      <Setter Property="BorderBrush" Value="#35516B" />
      <Setter Property="Padding" Value="12,8" />
    </Style>
  </ListBox.ItemContainerStyle>
</ListBox>

Use the selected row to drive detail content

private void OnServerListSelectionChanged(object? sender, SelectionChangedEventArgs e)
{
    if (ServerList.SelectedItem is ServerEntry selected)
    {
        DetailHost.Content = BuildServerPanel(selected);
    }
}

File picker with multi-select toggles

<ListBox Width="360"
         Height="220"
         SelectionMode="Multiple"
         ItemsSource="{Binding CandidateFiles}" />

Notes and pitfalls

Multiple selection is direct toggle mode. In the current implementation, clicking a row in SelectionMode="Multiple" toggles that row immediately. There is no WPF-style Extended selection mode with Ctrl or Shift gesture rules.

There is no separate SelectedItems collection. Use SelectedIndices to inspect the full multi-select state. SelectedIndex and SelectedItem expose the first selected entry.

Row visuals belong on ListBoxItem. If you only style the content inside the row, selected and hover states can feel disconnected from the clickable surface. Prefer an explicit ListBoxItem template or ItemContainerStyle.

Background and BorderBrush are colors, not brush objects. Keep examples and styles in terms of Color-compatible values.

Programmatic selection can move the viewport. Setting SelectedIndex or SelectedItem may scroll the internal viewer so the chosen row is fully visible.
