Controls

Label

Label is a lightweight text-display control in this framework. It inherits the rendering and layout behavior of TextBlock and adds a Padding dependency property for parser and theme compatibility, but it does not use that padding to change layout or rendering.

Quick start

Use Label when you want a named, styleable text caption element in XAML. It is well suited for headings, field captions, compact status text, and framework-generated fallback text content.

Minimal label

<Label Text="Display name" />

Styled caption

<Label Text="Diagnostics"
       Foreground="#D9E7F5"
       FontSize="20"
       FontWeight="Bold" />

Wrapped helper text

<Label Text="Changes are saved automatically after each successful sync."
       TextWrapping="Wrap"
       Width="240"
       Foreground="#AFC4D8" />

Default mental model: treat Label as a text primitive with a more semantic control name. If you need container chrome, inner spacing, or interactive label-target behavior, combine it with other controls instead of expecting those features from Label itself.

What Label is in this framework

This implementation is intentionally small. The class derives directly from TextBlock and only adds a dependency property named Padding.

What it is

A text-rendering element with the same core text pipeline as TextBlock, including Text, Foreground, typography inheritance, and optional wrapping.

What it is not

It is not a WPF-style content control, it does not host arbitrary child elements, and it does not provide a built-in target relationship to another control.

  • Label renders plain text from its Text property.
  • Its layout behavior comes from TextBlock.
  • The added Padding property exists for compatibility with theme setters and parsing scenarios.
  • The current class does not override measure, arrange, or render to apply padding.

Layout and rendering behavior

Because Label inherits directly from TextBlock, it follows the same text measurement and rendering rules.

Measure

If TextWrapping is NoWrap, the control sizes to the intrinsic width of the text. If wrapping is enabled, the available width constrains line breaking and the desired size grows vertically as needed.

Render

The control draws text lines directly at its layout slot using the resolved font settings, Foreground, and element Opacity.

  • Empty or null text produces a desired size of zero and renders nothing.
  • Wrapping only occurs when TextWrapping="Wrap".
  • Font family, size, weight, and style participate in both measurement and rendering.
  • Foreground controls text color.
<Border Background="#121B25"
        BorderBrush="#32485F"
        BorderThickness="1"
        Padding="12">
  <Label Text="This label wraps because the width is constrained by the surrounding layout."
         Width="220"
         TextWrapping="Wrap" />
</Border>

In that example, the surrounding Border contributes the inner spacing. The Label itself only measures and draws text.

The Padding property

Label exposes a Padding dependency property, but the current implementation does not use it during measure, arrange, or render.

Supported use

Accepting styles, setters, and XAML that assign Padding without failing. This is useful when sharing theme patterns across multiple control types.

Unsupported expectation

Do not expect Padding to add inner space around the text, change desired size, or shift the text rendering origin.

Accepted but visually inert padding

<Label Text="Theme-compatible label"
       Padding="12,6"
       Foreground="#D9E7F5" />

The XAML above is valid, and the property value is stored on the element. The current control still renders exactly like a TextBlock with the same text settings.

Practical recommendation: if you need visual spacing around label text, put the Label inside a container such as Border, Grid, or StackPanel and apply spacing there.

Differences from WPF Label

The name matches WPF, but the behavior does not fully match WPF's richer Label control.

This framework's Label

Text-only, derived from TextBlock, no arbitrary content model, no target property, and no built-in access-key host behavior.

Typical WPF expectation

A content-oriented control that can label another input element and often participates in mnemonic/access-key workflows.

  • Use Text, not arbitrary child content, as the primary label payload.
  • Do not expect a Target property on Label.
  • Do not expect Padding to create inner spacing.
  • If you need mnemonic rendering and Alt-key targeting, use AccessText where appropriate instead.

Use AccessText for mnemonic-style captions

<AccessText Text="_Save"
            TargetName="SaveButton" />

<Button x:Name="SaveButton"
        Content="Save now" />

Property reference

Primary text properties

Text: the string rendered by the control.

Foreground: a Color value for the rendered text.

TextWrapping: controls whether text stays on one line or wraps to multiple lines.

Inherited typography and layout

FontFamily, FontSize, FontWeight, and FontStyle: inherited text styling inputs used by the renderer.

Margin, Width, Height, alignment properties, and Opacity: inherited element-level behavior.

Padding: exposed on Label, but currently not consumed by layout or rendering.

Use color values that resolve to the framework's Color type for Foreground. When you need a background, border, or other decoration, pair the label with a container rather than looking for those properties on the label itself.

Patterns and examples

Form field caption

<StackPanel>
  <Label Text="Email address"
         Margin="0,0,0,6"
         Foreground="#D9E7F5" />
  <TextBox Width="280"
           Text="{Binding EmailAddress}" />
</StackPanel>

Section heading

<Label Text="Session Summary"
       FontSize="22"
       FontWeight="Bold"
       Foreground="#F4FAFF"
       Margin="0,0,0,10" />

Status text inside framed chrome

<Border Background="#1D3A2A"
        BorderBrush="#4BA36E"
        BorderThickness="1"
        Padding="8,4">
  <Label Text="ONLINE"
         Foreground="#DDF7E6" />
</Border>

Data-bound label

<Label Text="{Binding ActiveProfileName}"
       Foreground="#D9E7F5" />

Wrapped explanatory note

<Label Width="300"
       TextWrapping="Wrap"
       Foreground="#AFC4D8"
       Text="Selecting a preset applies typography, spacing, and color overrides to the current canvas." />

Style targeted at labels

<Style x:Key="FieldLabelStyle" TargetType="{x:Type Label}">
  <Setter Property="Foreground" Value="#C8D9EA" />
  <Setter Property="FontSize" Value="14" />
  <Setter Property="FontWeight" Value="SemiBold" />
  <Setter Property="Margin" Value="0,0,0,4" />
  <Setter Property="Padding" Value="6,2" />
</Style>

That final example is valid, but only the text and layout-related inherited properties affect rendering today. The Padding setter is accepted without changing the visual result.

Where Labels appear automatically

You may encounter Label even when you did not write one explicitly. Some framework presentation paths create labels as fallback text visuals for non-visual content.

ContentPresenter fallback

When a ContentPresenter receives plain data with no template and the content is not already a UIElement, it creates a Label to display that text.

Header fallback

HeaderedItemsControl can also generate a Label when a header is plain data and no header template is supplied.

Inherited styling

In those fallback cases, font settings and foreground are copied from the owning control, so the generated label visually matches the surrounding control styling.

Implicit text presentation through ContentPresenter

<ContentControl Content="Build succeeded" />

When no content template is involved, the presenter path can materialize a Label internally to show that string.

Notes and pitfalls

Padding is currently inert. It exists on the API surface, but it does not change the measured size or the rendered text position.

No arbitrary child content. This is not a general content host. Use a container or another control when you need nested visuals.

No target relationship. If you are looking for mnemonic or control-target behavior, Label is not the right surface in this framework.

Use containers for decoration. Backgrounds, borders, and true inner spacing belong on wrappers such as Border, not on Label.
